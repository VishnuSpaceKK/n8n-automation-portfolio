# Telegram Expense Tracker Bot (n8n)

A self-hosted n8n workflow that allows multiple users to log expenses through Telegram, stores them in Google Sheets, and automatically sends personalized weekly spending summaries.

## What it does

- Users message a Telegram bot with free-text expenses (e.g. "500 lunch 200 taxi").
- A JavaScript parser extracts each amount and description into separate expense entries.
- Each expense is stored as a separate row in Google Sheets with the user's `chat_id` and a timestamp.
- Every week, a scheduled workflow aggregates each user's last 7 days of expenses and automatically sends a personalized spending summary via Telegram.

## Architecture

### Workflow 1 — Expense Capture
1. Telegram Trigger
2. Parse free-text expenses (Code node)
3. Append each expense to Google Sheets

### Workflow 2 — Weekly Summary
1. Schedule Trigger
2. Read expenses from Google Sheets
3. Group expenses by `chat_id`
4. Calculate each user's 7-day total
5. Send a personalized Telegram summary

## Tech Stack

- **n8n** (self-hosted workflow automation)
- **Railway** (hosting via Docker)
- **Telegram Bot API**
- **Google Sheets API** (OAuth 2.0)
- **JavaScript** (n8n Code nodes)

## Deployment

- Deployed n8n as a Docker container on Railway.
- Configured a public HTTPS webhook endpoint for the Telegram Trigger.
- Stored Telegram bot credentials securely using n8n Credentials.
- Configured Google OAuth 2.0 through Google Cloud Console for Google Sheets access.

## Features

- Multi-user expense tracking
- Free-text expense parsing
- Automatic Google Sheets logging
- Weekly scheduled summaries
- Self-hosted n8n deployment

## Key Design Decisions

- **Single Google Sheet:** Stored all users' expenses in one sheet using a `chat_id` column instead of creating separate sheets for each user. This keeps the workflow simple and makes generating user-wise summaries easy.
- **Append-Only Data:** Never delete old expenses. Weekly reports are generated by filtering the last 7 days, allowing historical data to be retained for future analysis.
- **Support Multiple Expenses:** A single Telegram message (e.g. `"500 lunch 200 taxi"`) is automatically split into multiple expense entries before being stored.
